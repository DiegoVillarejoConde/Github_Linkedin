# Instrucciones

En este repositorio van a entregarse todas las tareas del módulo. Lee los siguientes apartados para realizar las entregas de forma adecuada

## Estructura

Las tareas las entregarás de forma estructurada. Para ello: 
- cada tema tendrá su propia carpeta (UD1, UD2...) y
- cada tarea tendrá su propia carpeta también UD2/tarea2_1/, UD2/tarea2_2/.
- cada carpeta contendrá al menos un README.md
  - el README.md del tema contendrá el título y un listado de enlaces a cada tarea
  - el README.md de la tarea contendrá las instruciones de la tarea dada en el moodle
  - las contestaciones se harán sobre el propio README, respetando un formato mínimo mediante el lenguaje Markdown

## Repositorio
Un repositorio contiene todos los archivos de tu proyecto y el historial de revisiones/commits de cada uno de ellos. Puedes debatir y administrar el trabajo de tu proyecto dentro del repositorio.

Acerca de la visibilidad de un repositorio, se puede restringir quién tiene acceso a un repositorio eligiendo la visibilidad de este: público o privado.

Este repositorio es privado y sólo compartido con la profesora.

# GitHub
GitHub es una plataforma de alojamiento de repositorios que ofrece colaboración y control de acceso. Es una herramienta de control de versiones para  trabajar de forma colaborativa sobre proyectos.

Algunas características de GitHub:

- Especifica hitos y etiquetas para los proyectos.
- GitHub Pages nos permite publicar y alojar sitios web dentro de GitHub.
- Se permite el trabajo con ramas
- Permite integraciones de API de terceros para el seguimiento de errores y alojamiento en la nube.

Ventajas:
 
- Nos ayuda a almacenar los datos en forma de metadatos.
- Se utiliza para compartir el trabajo de forma pública.

Desventajas:
 
- Almacenar archivos grandes en GitHub lo hace lento.
- Solo es compatible con el control de versiones de Git.

## Commit

Cada vez que realizas un git commit o arrastras directamente un fichero al repositorio de GitHub estás creando un commit, que es como una instantánea de tu repositorio. Estos commits son instantáneas de tu repositorio en momentos específicos. Se deben realizar nuevos commits con frecuencia, basadas en unidades lógicas de cambio. Con el tiempo, los commits deben contar la historia de su repositorio y cómo llegó a ser como es actualmente. Los commits incluyen muchos metadatos además del contenido y el mensaje, como el autor, la marca de tiempo...

## README.md
Cada repositorio debe tener un archivo README para comunicar información importante sobre su proyecto. Un README, junto con una licencia de repositorio, un archivo de citas, pautas de contribución y un código de conducta, comunica las expectativas de su proyecto y lo ayuda a administrar las contribuciones.

Un README es el primer elemento que un visitante verá cuando visite su repositorio. 


## Markdown

Markdown es un lenguaje de marcado ligero. Es un lenguaje que tiene la finalidad de permitir crear contenido de una manera sencilla de escribir, y que en todo momento mantenga un diseño legible, así que para simplificar puedes considerar Markdown como un método de escritura.

Este método  permitirá añadir formatos tales como negritas, cursivas o enlaces, utilizando simplemente texto plano, lo que hará de la escritura algo más simple y eficiente al evitar distracciones.

## Visual Studio Code

El editor de textos Visual Studio Code admite archivos Markdown listos para usar. Simplemente comience a escribir texto de Markdown, guarde el archivo con la extensión .md y luego podrá alternar la visualización del editor entre el código y la vista previa del archivo de Markdown; Obviamente, también se puede abrir un archivo Markdown existente y empezar a trabajar con él.

Para cambiar entre vistas, presione Ctrl+Shift+V en el editor. Puede ver la vista previa en paralelo (Ctrl+K V) con el archivo que está editando y ver los cambios reflejados en tiempo real a medida que edita.

Recuerda abrir LA CARPETA del repositorio, para trabajar SIEMPRE DESDE LOCAL.

## Cómo hacer las entregas en el repositorio

### Descarga el repositorio

Descarga este repositorio y trabaja SIEMPRE en local. Conforme vayas creando contenido en tu equipo local lo subirás al repositorio de GitHub para poder compartirlo.

Una vez descargado descomprime el fichero en el sitio adecuado. Desde el Visual Studio Code abre LA CARPETA del repositorio y actualízala con el contenido de tu tarea.
Recuerda abrir SIEMPRE la carpeta de trabajo para trabajar de forma más organizada.

### Agregar un archivo/carpeta a un repositorio

En GitHub.com, navega a la página principal del repositorio y arrastra la carpeta de la tarea. Debes escribir el mensaje del commit con el nombre de la tarea a entregar.
Recuerda arrastrar la CARPETA ENTERA para actualizar un fichero que no esté en el raíz del repositorio

### Entrega en el moodle
En la tarea del moodle recuerda entregar la url de la carpeta. Enlaza a la carpeta de la tarea en el repositorio. Se mostrará el README.md y desde ahí los enlaces a los ficheros entregados en su caso. Por ejemplo:
